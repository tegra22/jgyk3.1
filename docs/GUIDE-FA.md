## 🚀 معرفی پروژه

**MRH-G2Ray** یک نسخه پیشرفته و سطح بالا از پروژه `g2ray` است که از **GitHub Codespaces** (محیط‌های ابری رایگان توسعه) برای اجرای یک **پروکسی VLESS** روی زیرساخت جهانی گیت‌هاب استفاده می‌کند.

- تمام ترافیک از **سرورهای گیت‌هاب** عبور می‌کند (نیازی به VPS شخصی نیست)
- در مناطق دارای محدودیت اینترنت کار می‌کند (در صورت دسترسی به گیت‌هاب)
- معماری آماده تولید با **اسکریپت‌های ماژولار** و **مدیریت مبتنی بر وب**

## 📝 راهنمای گام به گام برای مبتدیان

### ۱. ساخت اکانت گیت‌هاب

| مرحله | اقدام |
|:---|:---|
| ۱ | به [GitHub.com](https://github.com) بروید |
| ۲ | روی **Sign up** کلیک کنید |
| ۳ | ایمیل، رمز قوی و نام کاربری وارد کنید |
| ۴ | کد تأیید ۶ رقمی را وارد کنید |

> ⚠️ **توصیه امنیتی:** از یک **اکانت دوم (فرعی)** استفاده کنید تا اکانت اصلی شما در معرض خطر قرار نگیرد.

### ۲. فورک کردن مخزن

1. به آدرس زیر بروید: `https://github.com/mrh000mrh/MRH-G2Ray-2`
2. روی دکمه **Fork** (بالا سمت راست) کلیک کنید
3. (اختیاری) نام مخزن را تغییر دهید (مثلاً `My-Proxy`)
4. روی **Create fork** کلیک کنید

### ۳. ساخت Codespace

1. در مخزن فورک‌شده، روی دکمه سبز **`Code`** کلیک کنید
2. برگه **`Codespaces`** را انتخاب کنید
3. روی **`Create codespace on main`** کلیک کنید
4. ۲ تا ۳ دقیقه منتظر بمانید تا محیط آماده شود

### ۴. دریافت لینک VLESS (دو روش)

#### روش الف: از طریق پنل مدیریت (توصیه می‌شود)

| مرحله | اقدام |
|:---|:---|
| ۱ | پس از بارگذاری کداسپیس، یک تب جدید به طور خودکار با پنل مدیریت باز می‌شود |
| ۲ | اگر باز نشد، به ترمینال → برگه **Ports** → روی آیکون کره (🌐) کنار پورت `8080` کلیک کنید |
| ۳ | با اطلاعات پیش‌فرض وارد شوید: `admin` / `Sample@Sample` |
| 4 | روی **`Copy VLESS Link`** کلیک کنید |

#### روش ب: از طریق ترمینال (جایگزین)

- لینکی که با `vless://` شروع می‌شود در خروجی ترمینال نمایش داده می‌شود
- کل لینک را دستی کپی کنید

> 🔑 **نکته امنیتی:** رمز پیش‌فرض پنل را بلافاصله تغییر دهید (به بخش تنظیمات پیشرفته مراجعه کنید).

### ۵. اتصال به اینترنت

## ⏰ مدیریت سهمیه: Stop در مقابل Delete

| عملیات | تأثیر بر ساعت | تأثیر بر فضا | بهترین کاربرد |
|:---|:---:|:---:|:---|
| **Stop** | ✅ متوقف می‌شود | ❌ ادامه می‌یابد | استراحت کوتاه (≤ ۱ ساعت) |
| **Delete** | ✅ متوقف می‌شود | ✅ متوقف می‌شود | بعد از هر بار استفاده یا استراحت طولانی |

- **Stop:** فقط ماشین مجازی را خاموش می‌کند. فایل‌های شما روی دیسک گیت‌هاب باقی می‌مانند و از سهمیه ۱۵ گیگابایتی شما کم می‌شوند. اگر قصد دارید ظرف چند ساعت دوباره کار کنید، از این گزینه استفاده کنید.
- **Delete:** کل محیط کداسپیس را حذف می‌کند. هم ساعت و هم فضای ذخیره‌سازی آزاد می‌شوند. **این گزینه پیش‌فرض شما برای حداکثر بهره‌وری باشد.**

> ✅ **گردش کار پیشنهادی:** بعد از هر بار استفاده، Codespace را **Delete** کنید و دفعه بعد دوباره بسازید. این کار تضمین می‌کند هیچ فضایی هدر نرود و مصرف سهمیه شما قابل پیش‌بینی باشد.

## 🛠️ تنظیمات پیشرفته (بخش حرفه‌ای)

### تغییر نوع ماشین (CPU و RAM)

برای افزایش منابع، فایل `.devcontainer/devcontainer.json` را ویرایش کنید:

```json
"hostRequirements": {
   "cpus": 4,
   "memory": "8gb",
   "storage": "64gb"
}



نوع ماشین	ساعت رایگان ماهانه
۲ هسته (پیش‌فرض)	۶۰ ساعت
۴ هسته	۳۰ ساعت
۸ هسته	۱۵ ساعت
۱۶ هسته	۷.۵ ساعت
امن‌سازی پنل مدیریت (تغییر رمز)
متغیرهای محیطی را در ترمینال کداسپیس تنظیم کنید:

export MRH_ADMIN_USERNAME="your-username"
export MRH_ADMIN_PASSWORD="your-strong-password"


یا آن‌ها را در .devcontainer/devcontainer.json ثابت کنید:

"containerEnv": {
   "MRH_ADMIN_USERNAME": "your-username",
   "MRH_ADMIN_PASSWORD": "your-strong-password"
}

همگام‌سازی با پروژه اصلی
برای دریافت به‌روزرسانی‌ها از مخزن اصلی:

به صفحه مخزن فورک‌شده خود در گیت‌هاب بروید

روی Sync fork کلیک کنید

روی Update branch کلیک کنید

نصب دستی (برای توسعه‌دهندگان)

# کلون کردن مخزن
git clone https://github.com/mrh000mrh/MRH-G2Ray-2.git
cd MRH-G2Ray-2

# ساخت Docker image
docker build -t mrh-g2ray .devcontainer/

# اجرای کانتینر
docker run -p 443:443 -p 8080:8080 mrh-g2ray



❓ عیب‌یابی و سوالات متداول
مشکل	راه‌حل
کداسپیس شروع نمی‌شود	سهمیه رایگان خود را بررسی کنید (Settings → Billing). اگر تمام شده، تا ماه بعد صبر کنید یا از اکانت دیگری استفاده کنید.
پنل مدیریت باز نمی‌شود	مطمئن شوید پورت 8080 در برگه "Ports" لیست شده است. اگر نیست، کداسپیس را مجدداً راه‌اندازی کنید.
رمز پیش‌فرض کار نمی‌کند	از admin / Sample@Sample استفاده کنید. بلافاصله بعد از اولین ورود آن را تغییر دهید.
لینک VLESS وصل نمی‌شود	مطمئن شوید لینک با vless:// شروع می‌شود. یک کلاینت دیگر (مثلاً Hiddify به جای V2rayNG) امتحان کنید.
خطای GitHub Actions	این خطا در اولین اجرا طبیعی است. ۲ دقیقه صبر کنید و صفحه را Refresh کنید. اگر ادامه داشت، مجدداً فورک کنید.
